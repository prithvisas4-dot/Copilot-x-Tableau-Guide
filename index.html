When to Use

Comparing values across discrete categories. Most effective for 2–20 categories. Use horizontal bars when labels are long.

Data Requirements

1+ dimensions, 1 measure. Add second dimension for stacked or side-by-side variants.

▶ Build This Chart

LIMITATION 01 — Cannot Mix Aggregate & Non-Aggregate Arguments
The Limitation

You cannot combine an aggregated value like SUM([Sales]) with a row-level value like [Sales] in the same calculated field. Error: "Cannot mix aggregate and non-aggregate arguments."

FreshMart Scenario

Trying to write SUM([Sales]) / [Sales] to calculate each product's share of total throws the error because the denominator is row-level while the numerator is aggregate.

Workaround

Use a FIXED LOD for the total: SUM([Sales]) / {FIXED : SUM([Sales])} — making both sides aggregate.

▶ Copilot Prompt
I'm getting this Tableau error: "Cannot mix aggregate and non-aggregate arguments." I was trying to calculate [describe your goal — e.g. each product's share of total sales]. Explain exactly why this error happens based on how Tableau separates aggregate and row-level calculation granularities, show me how to fix my specific scenario using a FIXED LOD expression with the correct full syntax, and explain when this error appears vs when I'd get a different error for the same mistake.
LIMITATION 02 — FIXED LOD Ignores Dimension Filters
The Limitation

FIXED LOD expressions are evaluated before dimension filters run. If a user filters to North region, a FIXED LOD still computes across all regions.

FreshMart Scenario

A FIXED LOD for company-wide total sales still shows the all-region total when a manager filters to just their region — making the % share calculation wrong.

Workaround

Promote the Region filter to a Context Filter, or rewrite as an INCLUDE/EXCLUDE LOD which runs after dimension filters.

▶ Copilot Prompt

Tell me which solution is better for my scenario and why.
LIMITATION 03 — Data Blending Only Supports Left Joins
The Limitation

When blending data sources, Tableau always left-joins the secondary source to the primary. Inner, right, and full outer joins are not available through the blending mechanism.

FreshMart Scenario

Blending POS sales (primary) with a targets Excel (secondary). Stores with no sales this week still appear with NULL targets — the left join keeps all primary rows even without a secondary match.

Workaround

Use a Relationship or physical Join in the Data Source page. If blending is unavoidable, wrap secondary fields with ZN() to replace NULLs.

LIMITATION 04 — Table Calculations Cannot Be Nested Inside LODs

Table calculations (RUNNING_SUM, RANK, LOOKUP, WINDOW_AVG, etc.) cannot be nested inside LOD expressions, and vice versa.

FreshMart Scenario

Compute the LOD as a standalone field, bring it into the view, then apply the table calculation on top of the rendered result — never nested in the same expression.

Section 02 · Copilot in Excel for Data Analysts

Excel Copilot understands your data table and lets you analyse, transform, visualise, and clean data using plain English. For Tableau analysts, this is most powerful for pre-processing exports before building dashboards and for quick ad-hoc analysis that doesn't need a full Tableau build.

📋 Pre-Requisites: How to Use Excel Copilot

Requirements

Microsoft 365 Copilot licence (separate from standard M365). Data must be in a named Excel Table — press Ctrl+T to convert a range. The Copilot button appears in the Home ribbon as a purple sparkle icon.

What Copilot Can Do in Excel

Write formulas from plain English descriptions. Add new calculated columns. Highlight or format cells based on conditions. Create PivotTables and charts. Identify trends and surface key insights. Clean and standardize data. Answer questions about the data.

What It Cannot Do

It cannot access data outside the current workbook unless you manually add it. It does not learn from previous sessions. Complex VBA or Power Query transformations may need to be done manually.
